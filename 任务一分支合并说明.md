# 任务一分支合并说明

本分支完成任务一：通道管理应用。该分支可以整体上传到 GitHub，后续由 `main` 合并。合并时建议保留任务一必要改动，并关注下方列出的测试辅助改动与潜在冲突文件。

## 任务一必要修改

- 新增通道契约：`ChannelInfo`、`HardwareStatus`、`InstrumentApplyResult`、`ChannelSettingsChangedMessage`、`IInstrumentService`。
- 扩展 `IDeviceDriver`，支持通道配置下发与查询。
- 新增 `DHDAS.Service.Signal/Instrument` 后端服务、领域规则、配置注册表和校验逻辑。
- 新增 `DHDAS.Plugin.ChannelManager` 插件，实现通道配置表、参数编辑、后端校验和下发。
- 在 Shell 中注册 `AddChannelManagerModule()`，让插件可以通过 DI 获取 `IInstrumentService`。
- 增加 Avalonia `DataGrid` 包和样式，用于任务一通道配置表。
- 增加 `IMessenger.Listen<T>()`，让通道参数变更消息可以被其他插件订阅。
- 更新 `SineWaveSimulator`，让模拟硬件支持 2048 通道配置、启用/禁用、增益、偏移和采样率验证。

## 测试辅助修改

- 当前默认流水线为 `AcquisitionService -> DataPushService`，用于任务一在本地模拟采集链路中验证通道配置。
- `Waveform` 插件当前作为多通道数据监视界面使用，用于验证通道启用/禁用和参数下发效果。
- `PluginBase`、`PluginManager`、主窗口相关文件增加了插件加载、卸载和生命周期能力，用于验证插件可插拔性。

## 合并注意事项

- 任务二或任务四合并时，可根据集成需要调整 `Program.cs` 中的 `pipelineScheme`，恢复或扩展为包含网络、存储节点的完整流水线。
- 任务三正式波形显示可以覆盖或替换当前 `Waveform` 插件中的临时多通道监视实现。
- `SineWaveSimulator` 的通道配置能力属于任务一验证链路的一部分，即使后续接入真实硬件，也建议保留作为本地调试后备。
