以下任务都要求从 **UI界面 -> ViewModel逻辑 -> 契约接口 -> 后端服务** 实现完整的闭环。

### 任务一：通道管理应用 (Channel Manager)
**负责人：开发 D（仪器控制工程师）**
**目标**：实现对硬件通道参数的实时配置与下发。

1.  **UI界面 (View)**：使用 `DataGrid` 实现通道配置表，包含：通道启用勾选、通道名称、增益(dB)、输入量程、采样率下拉框。
2.  **ViewModel逻辑**：实现参数修改的即时响应，当用户修改某一行参数时，自动触发后端校验。
3.  **契约定义 (Contract)**：
    *   定义 `ChannelInfo` 结构体：包含通道ID、物理量单位、增益、偏移量。
    *   定义 `IInstrumentService` 接口：包含 `ApplySettings(List<ChannelInfo>)` 和 `GetHardwareStatus()`。
4.  **后端服务 (Service)**：
    *   实现 **功能层**：负责接收 ViewModel 下发的配置，并调用驱动层模拟接口。
    *   实现 **领域层**：编写业务规则检查逻辑（如：增益不能超过100dB，采样率必须是硬件支持的固定频率）。
5.  **联动需求**：参数修改后，通过 `IMessenger` 发送全局消息，通知“曲线显示应用”更新坐标轴刻度。

### 任务二：工程管理应用 (Project Manager)
**负责人：开发 C（业务逻辑工程师）**
**目标**：建立实验数据的物理组织骨架。

1.  **UI界面 (View)**：实现“新建工程”对话框（输入工程名、存储路径选择）及主界面左侧的“工程任务树”。
2.  **ViewModel逻辑**：管理“当前活跃工程”的状态。点击“保存”按钮时，触发物理目录创建。
3.  **契约定义 (Contract)**：
    *   定义 `IProjectService` 接口：包含 `CreateProject(name, path)`、`OpenProject(path)`。
    *   定义 `IProjectContext`（全局单例）：存储当前工程的物理根路径，供存储服务使用。
4.  **后端服务 (Service)**：
    *   实现 **功能层**：负责磁盘目录的 IO 操作，并在工程目录下自动生成 `project.json` 元数据文件。
    *   实现 **领域层**：定义工程目录结构规范（如：`RawData` 文件夹、`Results` 文件夹、`Config` 文件夹）。
5.  **联动需求**：工程切换时，通知“存储服务”自动切换当前写入的目标文件句柄。

### 任务三：曲线显示应用 (Waveform Display)
**负责人：开发 B（高性能工程师）**
**目标**：实现 400MBps 级数据的高性能渲染。

1.  **UI界面 (View)**：引入基础模块的 `WaveformPainter`。支持多窗口平铺布局（1x1, 2x2, 2x4），支持鼠标滚轮对 Y 轴进行实时缩放。
2.  **ViewModel逻辑**：
    *   通过 `IDataPushService` 订阅特定通道的 `Session` 数据流。
    *   实现**降采样分发逻辑**：若数据量超过屏幕像素，进行抽稀处理，确保 UI 线程在 Windows 7 下维持 60FPS。
3.  **契约定义 (Contract)**：
    *   定义 `IDataPushService`：提供 `Subscribe(channelId)` 接口。
    *   确保 `RawDataPacket` 包含 `ActualLength` 字段。
4.  **后端服务 (Service)**：
    *   实现 **DataPushService**：作为管道节点运行，负责将 `GlobalRawPipe` 中的包根据 ID 分流到对应的 `SignalSession`。
    *   **内存规约**：强制执行 `using(refBuffer)`，确保 UI 线程读取完毕后，引用计数正确减量。
5.  **联动需求**：响应“通道管理应用”的名称修改，动态更新曲线的图例（Legend）。

### 任务四：分布式管理应用 (Distribution Manager)
**负责人：开发 A（系统架构师）**
**目标**：打通跨机数据传输链路，满足分布式计算需求。

1.  **UI界面 (View)**：实现“计算节点映射表”，包含：目标节点名称、IP地址、端口号、负责转发的通道范围（如：1-10号通道发给 A 电脑）。
2.  **ViewModel逻辑**：处理连接/断开指令，展示各网络链路的实时流量（MBps）和丢包统计。
3.  **契约定义 (Contract)**：
    *   定义 `INetworkService`：提供 `UpdateRoutingTable(List<NetworkRoute>)` 接口。
    *   定义 `NetworkDataPacket`：基于 MessagePack 的可序列化模型。
4.  **后端服务 (Service)**：
    *   实现 **NetworkSenderNode**：作为 `IPipelineNode` 插入流水线末端。
    *   实现 **异步 TCP 发送逻辑**：通过高性能 Buffer 队列，将 `RefCountBuffer` 序列化并推送至远端。
5.  **联动需求**：当网络断开时，通过 `IMessenger` 发送“网络异常”信号，让“工程管理应用”在 UI 上显示红灯警告。

### 本周共同遵循的“工业级”约束（验收红线）

*   **Win7 渲染兜底**：所有人必须在 Win11 开发机上测试，确认代码不使用任何 Win10 特有的库。
*   **DI 注册一致性**：各插件必须在各自的 `AddXXXModule` 扩展方法中完成所有 Service 和 IPipelineNode 的注册。
*   **异常处理**：每个 Service 的 `ExecuteAsync` 必须包含 `try-catch` 块，确保发生 Socket 错误或 IO 错误时，后台线程不卡死整个软件。
*   **日志埋点**：所有关键操作（如：连接驱动成功、创建工程成功、网络路由更新）必须调用基础设施层的日志接口记录 `Info` 级别日志。
