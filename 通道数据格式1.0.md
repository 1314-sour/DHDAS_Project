

1. **通道配置格式**：描述每个通道是谁、单位是什么、采样率/增益/量程是多少。
2. **通道数据格式**：描述实时采样数据如何在采集、显示、存储、网络转发之间流动。

建议统一如下。

**一、通道配置格式：`ChannelInfo`**

```csharp
public struct ChannelInfo
{
    public int ChannelId { get; set; }        // 内部编号：0-2047
    public bool IsEnabled { get; set; }       // 是否启用
    public string ChannelName { get; set; }   // 显示名称，如 CH0001
    public string Unit { get; set; }          // 物理量单位，如 V、mA、℃、N
    public double GainDb { get; set; }        // 增益，0-100 dB
    public double Offset { get; set; }        // 偏移量
    public string InputRange { get; set; }    // 输入量程，如 +/-10V
    public int SampleRate { get; set; }       // 采样率，如 1000、5000、10000
}
```

约定：

- 内部 `ChannelId` 使用 `0-2047`。
- UI 可以显示为 `1-2048`，但代码、存储、网络都用 `0-2047`。
- `ChannelName` 是曲线图例的来源。
- `Unit` 是曲线 Y 轴单位的来源。
- `SampleRate` 是波形 X 轴时间刻度的依据。
- `GainDb / Offset / InputRange` 是硬件下发和数据解释的依据。

**二、实时数据包格式：`RawDataPacket`**

```csharp
public struct RawDataPacket
{
    public long Timestamp { get; set; }       // 数据包起始时间，建议 DateTime.Now.Ticks
    public int ChannelId { get; set; }        // 对应 ChannelInfo.ChannelId
    public double SampleRate { get; set; }    // 当前数据包采样率
    public double[] Data { get; set; }        // 物理量数据
    public int ActualLength { get; set; }     // Data 中有效长度
}
```

约定：

- `Data` 是已经换算后的物理值，不是原始 ADC 码值。
- `ActualLength` 必须用于读取有效数据，不能盲目读完整数组。
- 波形显示、存储服务、网络转发都应该以这个结构为基础。
- 时间轴计算方式：

```text
第 i 个采样点时间 = Timestamp + i / SampleRate
```

**三、配置变更消息格式**

```csharp
public sealed class ChannelSettingsChangedMessage
{
    public DateTimeOffset Timestamp { get; }
    public IReadOnlyList<ChannelInfo> Channels { get; }
    public bool RefreshAxisScale { get; }
}
```

用途：

- 通道名称变化：任务三更新 Legend。
- 单位变化：任务三更新 Y 轴单位。
- 采样率变化：任务三更新 X 轴时间刻度。
- 增益/量程变化：任务三刷新坐标轴范围。
- 任务二可以收到后保存当前工程配置。

> 通道管理统一提供 `ChannelInfo` 作为 2048 通道元数据标准，实时采样统一使用 `RawDataPacket`，跨机传输统一使用与 `RawDataPacket` 等价的 `NetworkDataPacket`。所有模块用 `ChannelId(0-2047)` 关联配置、显示、存储和转发。
