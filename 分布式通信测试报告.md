# 分布式通信链路测试报告

## 1. 测试目标

验证当前阶段的分布式基础通信链路是否满足以下要求：

- 支持单机模拟 `1` 个发送端和 `2` 个接收端。
- 两个接收端可以通过不同监听端口同时启动，不发生端口冲突或存储文件占用。
- 发送端启动后，路由表默认为空，只有用户点击 `添加/应用路由` 后才加入路由。
- 发送端可以配置多个目标节点，包括节点名称、IP 地址、端口和负责转发的通道范围。
- 用户可以在路由表中选择指定路由，并将当前波形只发送到选中的目标节点。
- 路由表中的每条路由支持删除，防止误添加重复路由。
- 发送端可以生成指定通道的模拟正弦波，不同通道的测试波形应有明显区别。
- 发送端和接收端都可以在 `实时波形显示` 中查看对应波形。
- 接收端反馈包含来源节点、包编号、通道、长度和前几个采样值。
- 网络链路状态可以显示实时流量、成功次数、失败次数和丢包统计。
- 网络异常时通过前端弹窗反馈，不进行自动循环重试。

## 2. 测试环境

- 项目路径：`D:\DHDAS_Project`
- 运行框架：`.NET 6`
- 前端框架：`Avalonia`
- 测试方式：单机三开，使用 `127.0.0.1` 模拟一台发送设备和两台接收设备
- 接收端 A 监听端口：`5001`
- 接收端 B 监听端口：`5002`
- 发送端目标地址：`127.0.0.1`
- 路由 A：`CH0 -> 127.0.0.1:5001`
- 路由 B：`CH1 -> 127.0.0.1:5002`

## 3. 测试数据

发送端生成的测试数据为模拟正弦波：

- 采样率：`1000Hz`
- 数据长度：`1000` 点
- 波形长度：`1` 秒
- `CH0`：约 `5Hz`，幅值约 `1.00`
- `CH1`：约 `8Hz`，幅值约 `0.88`

说明：

- `CH0` 和 `CH1` 的波形周期数和幅值不同，便于在前端观察时区分。
- 如果测试更高通道号，通道号越大，测试波形频率越高，幅值略有降低。

## 4. 启动步骤

### 4.1 关闭旧进程

测试前先关闭所有已经运行的 `DHDAS.App.Shell` 窗口，避免 DLL 被占用或插件仍加载旧版本。

### 4.2 编译项目

在项目根目录执行：

```bash
dotnet build
```

预期结果：

```text
Build succeeded.
0 Warning(s)
0 Error(s)
```

### 4.3 启动接收端 A

打开第一个终端，执行：

```bash
dotnet run --project src/4_Shell/DHDAS.App.Shell/DHDAS.App.Shell.csproj -- --role receiver --listen-port 5001
```

预期前端表现：

- `分布式路由配置` 页面显示“接收端模式”。
- 页面提示接收端正在监听 `5001` 端口。
- 启动后弹窗提示接收端正在监听 `5001` 端口。
- 存储文件应为 `recorded_data_5001.dat`。

### 4.4 启动接收端 B

打开第二个终端，执行：

```bash
dotnet run --project src/4_Shell/DHDAS.App.Shell/DHDAS.App.Shell.csproj -- --role receiver --listen-port 5002
```

预期前端表现：

- `分布式路由配置` 页面显示“接收端模式”。
- 页面提示接收端正在监听 `5002` 端口。
- 启动后弹窗提示接收端正在监听 `5002` 端口。
- 存储文件应为 `recorded_data_5002.dat`。
- 接收端 B 不应因为 `recorded_data.dat` 被占用而启动失败。

### 4.5 启动发送端

打开第三个终端，执行：

```bash
dotnet run --project src/4_Shell/DHDAS.App.Shell/DHDAS.App.Shell.csproj -- --role sender
```

预期前端表现：

- `分布式路由配置` 页面显示“发送端模式”。
- `当前路由表` 初始为空。
- 页面显示“计算节点映射表”配置区，包含：
  - `节点名称`
  - `IP地址`
  - `端口`
  - `起始通道`
  - `结束通道`
- 页面显示以下操作项：
  - `添加/应用路由`
  - `测试通道`
  - `生成指定通道正弦波`
  - `发送到选中路由`
  - `连接选中路由`
  - `断开选中路由`
- 页面显示：
  - `当前路由表`
  - `网络链路状态（实时流量 / 丢包统计）`
- `当前路由表` 中每条路由右侧应显示 `X` 删除按钮。

## 5. 测试用例

### 5.1 路由表初始为空

操作步骤：

1. 启动发送端。
2. 进入 `分布式路由配置`。
3. 不点击 `添加/应用路由`。

预期结果：

- `当前路由表` 应为空。
- `网络链路状态（实时流量 / 丢包统计）` 应为空。
- 页面上方输入框可以有默认值，但默认值不应自动进入路由表。

### 5.2 添加两条接收端路由

操作步骤：

1. 在发送端填写第一条路由：
   - 节点名称：`Receiver-A`
   - IP地址：`127.0.0.1`
   - 端口：`5001`
   - 起始通道：`0`
   - 结束通道：`0`
2. 点击 `添加/应用路由`。
3. 填写第二条路由：
   - 节点名称：`Receiver-B`
   - IP地址：`127.0.0.1`
   - 端口：`5002`
   - 起始通道：`1`
   - 结束通道：`1`
4. 点击 `添加/应用路由`。

预期结果：

- `当前路由表` 显示两条路由：

```text
Receiver-A | 127.0.0.1:5001 | CH0-0 | 启用
Receiver-B | 127.0.0.1:5002 | CH1-1 | 启用
```

- `网络链路状态（实时流量 / 丢包统计）` 显示两条目标链路。
- 每条路由右侧均显示 `X` 删除按钮。

### 5.3 删除误添加路由

操作步骤：

1. 在发送端额外添加一条重复路由。
2. 在 `当前路由表` 中点击该重复路由右侧的 `X`。

预期结果：

- 被删除的路由从 `当前路由表` 中移除。
- 对应的链路状态从 `网络链路状态（实时流量 / 丢包统计）` 中移除。
- 后续发送不再命中被删除的路由。

### 5.4 切换页面后保留本次路由配置

操作步骤：

1. 完成 `5.2 添加两条接收端路由`。
2. 点击左侧 `实时波形显示`。
3. 再点击左侧 `分布式路由配置`。

预期结果：

- 本次启动期间已添加的两条路由仍显示在 `当前路由表` 中。
- 链路状态仍显示对应目标节点。
- 关闭发送端程序后重新启动，路由表可以为空，因为当前阶段只要求本次运行期间保留。

### 5.5 连接和断开选中路由

操作步骤：

1. 完成 `5.2 添加两条接收端路由`。
2. 在 `当前路由表` 中选中 `Receiver-A | 127.0.0.1:5001 | CH0-0`。
3. 点击 `连接选中路由`。
4. 观察前端弹窗和链路状态。
5. 点击 `断开选中路由`。

预期结果：

- 点击连接后，选中路由对应的链路状态显示为已连接。
- 点击断开后，选中路由对应的链路状态显示为未连接。
- 未选中的 `Receiver-B` 路由不应被连接或断开。

### 5.6 生成 CH0 波形并发送到 Receiver-A

操作步骤：

1. 在 `当前路由表` 中选中 `Receiver-A | 127.0.0.1:5001 | CH0-0`。
2. 确认 `测试通道` 自动变为 `0`。
3. 点击 `生成指定通道正弦波`。
4. 切换到发送端左侧 `实时波形显示`，确认显示 `CH0` 波形。
5. 回到 `分布式路由配置`。
6. 点击 `发送到选中路由`。

预期结果：

- 发送端弹窗提示已生成 `CH0` 波形，频率约 `5Hz`，幅值约 `1.00`。
- 发送端弹窗提示发送成功。
- 只有接收端 A 收到数据。
- 接收端 B 不应收到本次数据。
- 接收端 A 弹窗包含：

```text
来源节点: ...
包编号: ...
通道: 0
长度: 1000
前 8 个采样值: ...
```

- 接收端 A 的 `实时波形显示` 显示 `CH0` 波形。
- 发送端链路状态中 `127.0.0.1:5001` 成功次数增加。

### 5.7 生成 CH1 波形并发送到 Receiver-B

操作步骤：

1. 在 `当前路由表` 中选中 `Receiver-B | 127.0.0.1:5002 | CH1-1`。
2. 确认 `测试通道` 自动变为 `1`。
3. 点击 `生成指定通道正弦波`。
4. 切换到发送端左侧 `实时波形显示`，确认显示 `CH1` 波形。
5. 回到 `分布式路由配置`。
6. 点击 `发送到选中路由`。

预期结果：

- 发送端弹窗提示已生成 `CH1` 波形，频率约 `8Hz`，幅值约 `0.88`。
- 发送端弹窗提示发送成功。
- 只有接收端 B 收到数据。
- 接收端 A 不应收到本次数据。
- 接收端 B 弹窗包含：

```text
来源节点: ...
包编号: ...
通道: 1
长度: 1000
前 8 个采样值: ...
```

- 接收端 B 的 `实时波形显示` 显示 `CH1` 波形。
- `CH1` 波形应与 `CH0` 波形在周期数和幅值上有明显区别。
- 发送端链路状态中 `127.0.0.1:5002` 成功次数增加。

### 5.8 通道与选中路由不匹配

操作步骤：

1. 在 `当前路由表` 中选中 `Receiver-B | 127.0.0.1:5002 | CH1-1`。
2. 手动将 `测试通道` 改为 `0`。
3. 点击 `生成指定通道正弦波`。
4. 点击 `发送到选中路由`。

预期结果：

- 发送端弹窗提示通道与路由不匹配。
- 提示中应说明当前波形通道和选中路由负责的通道范围。
- 不应向 Receiver-B 发送 `CH0` 数据。
- 接收端 B 不应显示新的波形。

### 5.9 未生成数据直接发送

操作步骤：

1. 重启发送端。
2. 添加并选中一条路由。
3. 不点击 `生成指定通道正弦波`。
4. 直接点击 `发送到选中路由`。

预期结果：

- 前端弹窗提示没有可发送数据。
- 不应发生网络发送。
- 接收端不应显示新的波形。

### 5.10 接收端未启动时发送

操作步骤：

1. 关闭接收端 B。
2. 在发送端选中 `Receiver-B | 127.0.0.1:5002 | CH1-1`。
3. 生成 `CH1` 波形。
4. 点击 `发送到选中路由`。

预期结果：

- 发送端弹窗提示连接接收端失败或发送失败。
- 不应持续重试。
- 不应持续弹窗。
- 用户再次点击 `发送到选中路由` 时才会再次尝试发送。
- `网络链路状态（实时流量 / 丢包统计）` 中 `127.0.0.1:5002` 的失败次数增加。

### 5.11 多目标同通道分发

操作步骤：

1. 配置两条相同通道路由：
   - `Receiver-A | 127.0.0.1 | 5001 | CH0-0`
   - `Receiver-B | 127.0.0.1 | 5002 | CH0-0`
2. 生成 `CH0` 波形。
3. 验证后台按通道匹配多路由的分发能力。

预期结果：

- 两个接收端都可以收到 `CH0` 数据。
- 两个接收端弹窗中的 `包编号` 应相同，证明属于同一次分发。

说明：

- 当前前端主流程以 `发送到选中路由` 为主。
- 多目标同通道分发用于验证后台按通道匹配多路由的能力。

### 5.12 发送端后台节点持续运行验证

操作步骤：

1. 启动发送端后等待 `30` 秒。
2. 添加并选中一条路由。
3. 生成对应通道波形。
4. 点击 `发送到选中路由`。

预期结果：

- 发送端不应出现“发送队列已满”。
- 发送端后台节点不应在启动后自动退出。
- 如果仍出现发送队列不可用提示，应检查发送端后台节点是否仍在运行。
- 正常情况下应完成一次发送。

## 6. 验收标准

本阶段验收通过条件：

- 发送端和接收端界面能通过 `--role sender` / `--role receiver` 明确区分。
- 两个接收端可以通过 `--listen-port 5001` 和 `--listen-port 5002` 在同一台机器上同时启动。
- 两个接收端使用不同存储文件，不应互相占用 `recorded_data.dat`。
- 发送端启动后路由表默认为空。
- 用户手动添加路由后，路由表可以显示目标节点、IP、端口和通道范围。
- 路由表切换页面后，在本次程序运行期间可以保留。
- 路由表中的路由可以被删除。
- 用户可以选中某一条路由，并只向该路由对应的接收端发送数据。
- `CH0` 和 `CH1` 可以生成不同形态的测试波形。
- `CH0` 可以只发送到接收端 A，`CH1` 可以只发送到接收端 B。
- 接收端可以通过弹窗看到来源节点、包编号、通道和数据长度。
- 接收端收到后，能在 `实时波形显示` 中看到完整波形。
- 网络链路状态可以显示实时流量、成功次数、失败次数和丢包统计。
- 通信成功、失败、接收成功等反馈通过前端弹窗显示。
- 网络异常时不会持续轮询重试。
- 发送端后台节点应持续运行，不应因没有上游输入而提前关闭发送队列。

## 7. 注意事项

- 如果修改代码后运行时界面没有变化，先关闭所有 `DHDAS.App.Shell` 窗口，再重新执行 `dotnet build`。
- 插件 DLL 被运行中的 Shell 占用时，构建可能失败，需要关闭旧窗口后重试。
- 如果第二个接收端启动失败并提示文件被占用，应确认当前版本已经使用 `recorded_data_5001.dat`、`recorded_data_5002.dat` 这种端口后缀文件。
- 当前路由配置只要求在本次程序运行期间保留，关闭发送端后不要求自动恢复。
- 如果点击发送时提示“发送队列不可用”，优先检查是否运行了旧版本，或发送端后台节点是否已退出。
- 临时目录如 `tmp-build`、`tmp-build-plugin`、`tmp-build-waveform`、`tmp-build-shell` 不是项目源码，可以删除。
