# 分布式通信链路测试报告

## 1. 测试目标

验证当前阶段的分布式基础通信链路是否满足以下要求：

- 发送端可以生成一段模拟正弦波数据。
- 发送端可以在前端“实时波形显示”中看到待发送波形。
- 发送端可以通过“计算节点映射表”配置目标节点名称、IP 地址、端口和通道范围。
- 发送端可以展示当前路由表，并将路由表同步到后台网络服务。
- 发送端可以展示网络链路状态，包括实时流量、成功次数、失败次数和丢包统计。
- 用户确认后，由发送端手动发送当前波形。
- 接收端可以正确接收该波形数据。
- 接收端可以在前端“实时波形显示”中看到接收到的完整波形。
- 收发反馈通过前端弹窗展示，不依赖控制台日志作为主要反馈。
- 网络异常时通过前端弹窗反馈，不进行自动循环重试。

## 2. 测试环境

- 项目路径：`D:\DHDAS_Project`
- 运行框架：`.NET 6`
- 前端框架：`Avalonia`
- 测试方式：单机双开，使用 `127.0.0.1` 模拟两台设备
- 接收端监听端口：`5000`
- 发送端默认目标：`127.0.0.1:5000`
- 默认路由通道范围：`CH0-0`

## 3. 测试数据

发送端生成的测试数据为模拟正弦波：

- 通道：`CH0`
- 采样率：`1000Hz`
- 数据长度：`1000` 点
- 频率：`5Hz`
- 波形长度：`1` 秒
- 完整周期数：`5` 个周期

## 4. 启动步骤

### 4.1 关闭旧进程

测试前先关闭所有已经运行的 `DHDAS.App.Shell` 窗口，避免 DLL 被占用或插件仍加载旧版本。

### 4.2 编译项目

在项目根目录执行：

```bash
dotnet build
```

预期结果：

```text
Build succeeded.
0 Warning(s)
0 Error(s)
```

### 4.3 启动接收端

打开第一个终端，执行：

```bash
dotnet run --project src/4_Shell/DHDAS.App.Shell/DHDAS.App.Shell.csproj -- --role receiver
```

预期前端表现：

- Shell 左侧仍显示插件列表。
- `分布式路由配置` 页面显示“接收端模式”。
- 页面只显示接收端说明，不显示发送端路由配置和发送按钮。
- 启动后弹窗提示接收端正在监听 `5000` 端口。

### 4.4 启动发送端

打开第二个终端，执行：

```bash
dotnet run --project src/4_Shell/DHDAS.App.Shell/DHDAS.App.Shell.csproj -- --role sender
```

预期前端表现：

- `分布式路由配置` 页面显示“发送端模式”。
- 页面显示“计算节点映射表”配置区，包含：
  - `节点名称`
  - `IP地址`
  - `端口`
  - `起始通道`
  - `结束通道`
- 页面显示以下操作按钮：
  - `添加/应用路由`
  - `生成测试正弦波 (CH0)`
  - `发送当前波形`
  - `连接第一条路由`
  - `断开第一条路由`
- 页面显示：
  - `当前路由表`
  - `网络链路状态（实时流量 / 丢包统计）`

## 5. 测试用例

### 5.1 配置计算节点映射表

操作步骤：

1. 在发送端选择 `分布式路由配置`。
2. 在映射表中填写：
   - 节点名称：`本地回环节点`
   - IP地址：`127.0.0.1`
   - 端口：`5000`
   - 起始通道：`0`
   - 结束通道：`0`
3. 点击 `添加/应用路由`。

预期结果：

- `当前路由表` 中显示一条路由。
- 路由内容应包含：

```text
本地回环节点 | 127.0.0.1:5000 | CH0-0 | 启用
```

- `网络链路状态（实时流量 / 丢包统计）` 中出现对应链路状态。
- 初始状态中成功次数、失败次数应为 `0`。

### 5.2 连接和断开第一条路由

操作步骤：

1. 完成 `5.1 配置计算节点映射表`。
2. 点击 `连接第一条路由`。
3. 观察前端弹窗和链路状态。
4. 点击 `断开第一条路由`。

预期结果：

- 点击连接后，前端弹窗提示连接成功或连接异常。
- 接收端已启动时，链路状态应显示为已连接。
- 点击断开后，链路状态应显示为已断开。
- 连接或断开操作不应导致界面卡死。

### 5.3 生成发送端波形

操作步骤：

1. 在发送端选择 `分布式路由配置`。
2. 点击 `生成测试正弦波 (CH0)`。
3. 切换到发送端左侧的 `实时波形显示`。

预期结果：

- 发送端弹窗提示已生成波形。
- `实时波形显示` 页面显示一段完整正弦波。
- 波形应包含约 5 个完整周期。
- 标题显示类似：

```text
通道 0 | 长度 1000 | 最新接收波形
```

### 5.4 未生成数据直接发送

操作步骤：

1. 重启发送端。
2. 不点击 `生成测试正弦波 (CH0)`。
3. 直接点击 `发送当前波形`。

预期结果：

- 前端弹窗提示没有可发送数据。
- 不应发生网络发送。
- 接收端不应显示新的波形。

### 5.5 正常发送当前波形

操作步骤：

1. 在发送端完成 `5.1 配置计算节点映射表`。
2. 点击 `生成测试正弦波 (CH0)`。
3. 在发送端 `实时波形显示` 中确认波形已显示。
4. 返回发送端 `分布式路由配置`。
5. 点击 `发送当前波形`。

预期结果：

- 发送端弹窗提示发送成功。
- 接收端弹窗提示收到数据。
- 接收端弹窗包含：

```text
通道: 0
长度: 1000
前 8 个采样值: ...
```

- `网络链路状态（实时流量 / 丢包统计）` 中成功次数增加。
- 实时流量应出现非零统计值。

### 5.6 接收端显示完整波形

操作步骤：

1. 完成 `5.5 正常发送当前波形`。
2. 切换到接收端左侧的 `实时波形显示`。

预期结果：

- 接收端显示发送端发来的完整正弦波。
- 波形应包含约 5 个完整周期。
- 数据长度应显示为 `1000`。
- 波形不应只显示一个点或一小段曲线。

### 5.7 接收端未启动时发送

操作步骤：

1. 关闭接收端。
2. 在发送端完成 `5.1 配置计算节点映射表`。
3. 点击 `生成测试正弦波 (CH0)`。
4. 点击 `发送当前波形`。

预期结果：

- 发送端弹窗提示连接接收端失败或发送失败。
- 不应持续重试。
- 不应持续弹窗。
- 用户再次点击 `发送当前波形` 时才会再次尝试发送。
- `网络链路状态（实时流量 / 丢包统计）` 中失败次数增加。

### 5.8 未配置匹配路由时发送

操作步骤：

1. 启动发送端。
2. 将路由通道范围填写为：
   - 起始通道：`1`
   - 结束通道：`10`
3. 点击 `添加/应用路由`。
4. 点击 `生成测试正弦波 (CH0)`。
5. 点击 `发送当前波形`。

预期结果：

- 前端弹窗提示未配置发送路由或没有匹配 `CH0` 的路由。
- 不应向接收端发送数据。
- 接收端不应显示新的波形。

### 5.9 发送端后台节点持续运行验证

操作步骤：

1. 启动发送端后等待 `30` 秒。
2. 完成 `5.1 配置计算节点映射表`。
3. 点击 `生成测试正弦波 (CH0)`。
4. 点击 `发送当前波形`。

预期结果：

- 发送端不应出现“发送队列已满”。
- 发送端后台节点不应在启动后自动退出。
- 如果仍出现发送队列不可用提示，应检查发送端后台节点是否仍在运行。
- 正常情况下应按 `5.5 正常发送当前波形` 的结果完成发送。

## 6. 验收标准

本阶段验收通过条件：

- 发送端和接收端界面能通过 `--role sender` / `--role receiver` 明确区分。
- 发送端可以通过计算节点映射表配置目标节点名称、IP 地址、端口和通道范围。
- 当前路由表可以显示已配置的路由信息。
- 网络链路状态可以显示实时流量、成功次数、失败次数和丢包统计。
- 发送端不会自动循环发送数据。
- 发送端必须先生成波形，才能发送当前波形。
- 发送端生成的波形能在发送端 `实时波形显示` 中看到。
- 接收端收到后，能在接收端 `实时波形显示` 中看到同一段完整波形。
- 通信成功、失败、接收成功等反馈通过前端弹窗显示。
- 网络异常时不会持续轮询重试。
- 发送端后台节点应持续运行，不应因没有上游输入而提前关闭发送队列。

## 7. 注意事项

- 如果修改代码后运行时界面没有变化，先关闭所有 `DHDAS.App.Shell` 窗口，再重新执行 `dotnet build`。
- 插件 DLL 被运行中的 Shell 占用时，构建可能失败，需要关闭旧窗口后重试。
- 临时目录如 `tmp-build`、`tmp-build-plugin`、`tmp-build-waveform` 不是项目源码，可以删除。
- 如果点击发送时提示“发送队列不可用”，优先检查是否运行了旧版本，或发送端后台节点是否已退出。
